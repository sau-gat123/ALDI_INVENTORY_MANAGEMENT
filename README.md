# 🏬 Inventory Management System (PostgreSQL)

## 📌 Overview

This project demonstrates the design and implementation of a **centralized inventory management system** for a retail business.

The goal is to simulate how a high-volume retailer can transition from manual tracking to a **structured relational database system**, enabling efficient inventory control, transaction tracking, and business insights.

---

## 🎯 Objectives

- Design a **normalized relational database (3NF)**
- Track **inventory across multiple locations**
- Record **inbound and outbound transactions**
- Enable **analytical queries** for business insights
- Build a **scalable and maintainable schema**

---

## 🧱 Database Design

### 🔹 Entities

- **Supplier** – Stores supplier information
- **Location** – Represents warehouses, stores, and distribution centers
- **Product** – Contains product details
- **Inventory** – Tracks stock levels per location
- **Inventory_Transaction** – Records stock movement (IN / OUT)
- **Transaction_Detail** – Stores product-level transaction data
